# 위임 체크리스트

## 풀스택 작업 시 (백엔드 + 프론트엔드 동시)

### 1. API 계약서 작성
- 모든 엔드포인트의 **정확한 응답 JSON 예시**를 양쪽에 전달한다.
- 필드명, 타입, 중첩 구조를 명시한다.
- "이런 느낌"이 아니라 **실제 JSON**을 준다.

### 2. 작업 순서
- 가능하면 **백엔드 먼저** 완료 → API 응답 확인 → 프론트엔드 시작
- 동시에 시킬 경우, API 계약서를 100% 확정한 후 시작한다.

### 3. 완료 후 검증
- 백엔드 완료 시: API 응답을 직접 `curl`로 확인한다.
- 프론트엔드 완료 시: 실제 API와 연동 테스트한다.
- **양쪽 필드명이 일치하는지** 반드시 확인한다.

### 4. 배포 전 체크
- `.env` 파일에 localhost가 없는지 확인
- `npm run build` 성공 확인
- 테스트 통과 확인
- 빌드 결과물 배포 후 실제 접속 테스트

### 5. 실수 발견 시
- 원인을 정확히 파악한다.
- 재발 방지책을 스킬/규칙에 추가한다.
- 사장님에게 솔직하게 보고한다.
